# AGENTS.md

## Introduction

Welcome, fellow agents! This document provides a comprehensive guide to working on this project. Please read it carefully and adhere to the guidelines to ensure a smooth and efficient workflow.

## Project Overview

This project is a simple, yet powerful, demonstration of the capabilities of AI-powered coding assistants. It consists of a single HTML page that displays a "Hello, world!" message. While the project itself is simple, the underlying principles and the potential for expansion are vast.

## Your Role as an Agent

As an agent, your primary role is to assist in the development and maintenance of this project. This includes, but is not limited to:

*   **Writing code:** You will be responsible for writing clean, efficient, and well-documented code.
*   **Testing code:** You must thoroughly test your code to ensure that it is free of bugs and meets the project requirements.
*   **Debugging code:** You will be responsible for identifying and fixing bugs in the code.
*   **Improving the project:** You are encouraged to suggest and implement improvements to the project.

## Coding Conventions

To maintain a consistent and readable codebase, please adhere to the following coding conventions:

*   **HTML:**
    *   Use semantic HTML5 tags.
    *   Indent your code with two spaces.
    *   Use lowercase for all tags and attributes.
    *   Use double quotes for all attribute values.
*   **CSS:**
    *   Use a separate CSS file for all styles.
    *   Use a consistent naming convention for all classes and IDs.
    *   Use a linter to ensure that your CSS is well-formed.
*   **JavaScript:**
    *   Use a separate JavaScript file for all scripts.
    *   Use a consistent coding style (e.g., Airbnb, Google).
    *   Use a linter to ensure that your JavaScript is well-formed.

## Testing

All code must be thoroughly tested before it is submitted. This includes:

*   **Unit tests:** Each function and module must have its own unit test.
*   **Integration tests:** The project must have a suite of integration tests that verify that all of the components work together as expected.
*   **End-to-end tests:** The project must have a suite of end-to-end tests that verify that the entire application works as expected.

## Submitting Your Work

When you are ready to submit your work, please follow these steps:

1.  **Create a new branch:** Create a new branch for your changes.
2.  **Commit your changes:** Commit your changes with a clear and descriptive commit message.
3.  **Create a pull request:** Create a pull request to merge your changes into the main branch.
4.  **Request a review:** Request a review from one of the project maintainers.

## Conclusion

Thank you for your contributions to this project. Your hard work and dedication are greatly appreciated. By following these guidelines, we can work together to create a high-quality and successful project.
